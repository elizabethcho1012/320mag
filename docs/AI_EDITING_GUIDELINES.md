# Third Twenty AI 에디팅 자동화 시스템 가이드라인

## 📋 목적
해외 트렌드 기사를 수집하여 한국 시니어 독자(40-60대)를 위한 **완전히 새로운 콘텐츠**로 재창작하는 AI 에디팅 시스템의 법적·윤리적 기준을 정립합니다.

**최종 업데이트**: 2025-12-07
**시스템 구성**: GPT-4 (카테고리 추론) + Claude 3.5 Haiku (콘텐츠 리라이팅)
**월간 비용**: ₩2,430 (90개 기사)
**매거진 형식**: 1,500-2,000자 본문 + 별도 요약글 + 이미지 3-5장

---

## ⚖️ 법적 기준

### 1. 저작권 침해 방지
- ❌ **금지**: 원문 복사·붙여넣기
- ❌ **금지**: 단순 번역 (번역물도 저작권 보호 대상)
- ❌ **금지**: 핵심 문장/표현의 직접적 사용
- ✅ **허용**: 사실 정보 활용 (저작권 보호 대상 아님)
- ✅ **허용**: 완전히 새로운 관점과 표현으로 재작성

### 2. Fair Use 원칙 준수
미국 저작권법의 Fair Use 4가지 요소:
1. **목적과 성격**: 상업적 목적이지만 **변형적 사용**(transformative use)
   - 원문: 일반 독자 대상
   - 우리: 한국 시니어 독자 맞춤 콘텐츠
2. **원저작물의 성격**: 뉴스/사실 기반 (보호 수준 낮음)
3. **사용된 양과 중요성**: 사실만 발췌, 표현은 전혀 다르게
4. **시장 영향**: 다른 시장/독자층 (경쟁 관계 없음)

### 3. 출처 표기
- ✅ **필수**: 원문 URL을 `source_url`에 저장
- ✅ **필수**: "해외 트렌드 기반 재창작" 명시
- ✅ **권장**: 기사 하단에 "참고: [원문 제목](URL)" 표기

## 🎯 콘텐츠 변형 기준 (Transformation Standards)

### 🚨 핵심 원칙: 사실 기반 저널리즘 (2025-12-07 추가)

**매거진은 사실을 다룹니다. 가상의 이야기를 만들지 않습니다.**

#### ✅ 허용되는 것:
- 원문의 사실 정보를 정확히 유지
- 인물명, 이벤트명, 제품명, 날짜, 장소 등 **정확한 정보 유지**
- 사실을 바탕으로 에디터만의 감각적인 글로 재구성
- 전문가 관점의 해석과 조언 추가

#### ❌ 금지되는 것:
- 없는 인물이나 이벤트 창작
- 사실 정보 변경 (예: Anya Taylor-Joy → Tom Stoppard)
- 주제를 완전히 다른 것으로 변경
- 가상의 일화나 대화 만들기

**핵심**: 리라이팅은 "새로운 이야기 만들기"가 아니라 **"사실을 새로운 시각으로 재구성"**입니다.

---

### Level 1: 사실 추출 ✅
원문에서 **사실 정보만** 추출:
- WHO: 누가
- WHAT: 무엇을
- WHEN: 언제
- WHERE: 어디서
- WHY: 왜

**예시:**
```
원문: "Tommy Hilfiger received the Outstanding Achievement Award at the Fashion Awards 2024."
추출: Tommy Hilfiger, Outstanding Achievement Award, Fashion Awards 2024
```

⚠️ **중요**: 사실 정보는 정확히 유지해야 합니다. 다른 인물이나 이벤트로 바꾸지 마세요!

### Level 2: 관점 전환 ✅ (가장 중요!)
한국 시니어 독자 맞춤 관점으로 **완전히 재해석**:

#### 나쁜 예 ❌ (단순 번역)
```
"Tommy Hilfiger가 2024 패션 어워드에서 공로상을 수상했습니다."
```
→ 원문 구조 그대로, 번역만 함

#### 좋은 예 ✅ (변형적 재창작)
```
"패션계의 거장 Tommy Hilfiger, 50년 경력 인정받아
40-60대에게 전하는 메시지: '나이는 창의성의 적이 아니다'

패션 산업에서 반세기를 보낸 Tommy Hilfiger가 런던 패션 어워드에서
평생 공로상을 받았습니다. 그의 여정은 우리 세대에게 묻습니다.
'당신의 50년은 무엇을 쌓아왔습니까?'..."
```
→ 시니어 독자 관점, 새로운 서사, 인생 교훈 강조

### Level 3: 에디터 페르소나 적용 ✅
각 AI 에디터의 전문성과 어조 반영:

**Sophia (패션 에디터) 스타일:**
```
"에이지리스 패션의 진정한 의미를 보여주는 순간이었습니다.
Hilfiger의 수상은 단순한 업적 인정이 아닙니다.
나이 들수록 더 빛나는 스타일이 있다는 증명입니다..."
```

**Jane (뷰티 전문의) 스타일:**
```
"피부과 전문의로서 주목한 점은 이번 블랙프라이데이
뷰티 트렌드가 '안티에이징'에서 '헬시에이징'으로 전환되고 있다는 것입니다..."
```

### Level 4: 로컬라이제이션 ✅
한국 문화와 시니어 독자에게 익숙한 맥락 추가:

```
원문: "The restaurant offers Mediterranean cuisine"
→ "지중해식 식단은 한국의 '약식동원(藥食同源)' 철학과 통합니다.
    올리브오일, 신선한 채소, 생선 중심의 식사는..."
```

## ⚠️ 위험 신호 (Red Flags)

다음 패턴이 보이면 즉시 프롬프트 수정 필요:

### 법적 위험
1. **번역기 어투**: "~했습니다", "~것입니다" 반복
2. **사실 나열**: 원문 순서대로 정보만 전달
3. **짧은 분량**: 1000자 미만 (변형 부족)
4. **원문 구조**: 도입-본문-결론이 원문과 동일
5. **독자 맞춤 부족**: 시니어 관점 없음

### AI 티 나는 표현 (독자 신뢰 하락)
1. **로봇 같은 완벽함**: 모든 문장이 정확한 문법, 균일한 길이
2. **감정 부재**: "객관적으로", "데이터에 따르면" 등 차가운 표현
3. **천편일률적 구조**: 모든 기사가 똑같은 패턴
4. **과도한 정리**: "첫째, 둘째, 셋째" / "결론적으로" 남발
5. **개성 없음**: 에디터별 차이가 없음
6. **일화 부족**: 개인적 경험이나 구체적 사례 없음
7. **AI 자기언급**: "분석 결과", "연구에 따르면" (출처 없이)

### 인간다움 체크리스트
- [ ] 문장 길이가 다양함 (짧고 긴 문장 섞임)
- [ ] 감탄사나 추임새 포함됨
- [ ] 독자에게 직접 말 걸기 ("여러분은 어떠신가요?")
- [ ] 구어체 표현 가끔 사용
- [ ] 개인적 의견/경험 명확히 드러남
- [ ] 에디터 페르소나가 뚜렷함 (Sophia와 Jane의 어투가 확연히 다름)

## 📸 이미지-콘텐츠 매칭 기준 (2025-12-07 개선)

### 매거진 이미지 표준
**업계 조사 결과** ([참고](https://wordcounter.net/blog/2016/02/04/101200_publisher-word-count.html)):
- **짧은 기사 (1000자)**: 1-2장
- **중간 기사 (1500-2000자)**: 3-4장 ⭐ **현재 적용**
- **긴 피처 (2000자+)**: 4-6장

**현재 구현**:
- **메인 이미지 1장**: RSS/OG Image/Unsplash
- **추가 이미지 3-5장**: AI가 본문에 어울리는 이미지 설명 제안 (선택)

### 문제점
원본 기사 이미지를 사용하지만, AI가 리라이팅하면서 주제를 완전히 바꾸는 경우 발생:
- 원본: "Anya Taylor-Joy의 드레스" + 이미지(Anya 사진)
- 리라이팅: "Tom Stoppard 연극" (완전히 다른 주제)
- 결과: 이미지 ≠ 내용 ❌

### 해결책

#### 1. 이미지 우선 순위
```
1순위: RSS 원본 이미지 (가장 정확)
2순위: OG Image (원본 기사 URL에서 추출)
3순위: Unsplash (원본 영문 제목으로 검색)
```

#### 2. 리라이팅 제약
이미지가 있는 경우 AI에게 명확한 제약:
- ✅ 원본 기사의 핵심 주제 유지 필수
- ✅ 인물명/브랜드명 변경 금지
- ✅ 주제를 완전히 다른 것으로 바꾸지 않기

**AI 프롬프트 예시:**
```
⚠️ 중요: 이 기사에는 원본 이미지가 있습니다!
원본 기사의 핵심 주제와 인물을 유지하세요.

잘못된 예: 원본 "Anya Taylor-Joy 드레스" → "Tom Stoppard 연극" ❌
올바른 예: 원본 "Anya Taylor-Joy 드레스" → "아냐 테일러조이의 레드카펫 패션" ✅
```

#### 3. 추가 이미지 제안 (신규)
AI가 본문 내용에 어울리는 추가 이미지 3-5개의 설명을 생성:
```
예시:
1. "60대 여성의 우아한 코트 스타일링"
2. "마라케시 영화제 레드카펫 전경"
3. "시니어 패션 아이콘들의 모습"
```
→ 실제 이미지는 Unsplash API로 검색하여 적용 (향후 구현)

#### 3. Unsplash 폴백 개선
- ❌ **이전**: 리라이팅된 한국어 제목으로 검색 → 부정확
- ✅ **개선**: **원본 영문 제목**으로 검색 → 정확한 이미지

**코드 구현:**
```typescript
// Before
featuredImageUrl = getSmartUnsplashUrl(
  rewritten.title,  // 한국어 제목
  rewritten.content,
  category
);

// After
featuredImageUrl = getSmartUnsplashUrl(
  article.title,     // 원본 영문 제목
  article.content,   // 원본 영문 내용
  category
);
```

#### 4. 품질 검증
매거진 신뢰성을 위한 원칙:
- 실존 인물 → 그 사람의 사진 필수
- 특정 제품/브랜드 → 해당 제품 이미지 필수
- 이벤트 → 해당 이벤트 사진 필수

**이미지가 부정확하면 차라리 제거하는 것이 낫습니다.**

4. **이미지 제안** (선택): 본문에 어울리는 추가 이미지 3-5개 설명
